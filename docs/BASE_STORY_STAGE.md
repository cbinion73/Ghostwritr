# Base Story Instruction Set
## Narrative Synthesis Stage - Operational

---

## PHASE OVERVIEW

After TOC is locked, Base Story stage generates the complete narrative arc of the book.

Base Story creates the narrative skeleton that research, stories, and detailed prose will later attach to.

---

## TRIGGER

TOC phase locked as v1.0 FINAL (entire Outline Stage complete).

Author clicks "Generate Base Story" or system auto-triggers.

---

## GHOSTWRITER TASK

**Input:**
- Knowledge Base (all locked research, personas, truth claim, transformation journey, voice blend from Promise Stage)
- Locked TOC (all sections, chapters, paragraphs with purposes and word counts)
- Locked Book Pitch (complete strategic synthesis)
- Writing Constitution (core principles, style guidelines, quality control standards)

**Process:**
Write complete narrative arc following outline, thinking through:
- The book's parable (deeper story beneath the framework)
- Reader journey (5-phase transformation: Current Reality → Disruption → Revelation → Application → Transformation)
- Themes (recurring ideas that weave throughout)
- Tension architecture (where discomfort builds, peaks, resolves)
- Persona journeys (how all three personas experience this differently)
- Voice blend shifts (where tone/approach changes)
- Permission points (where readers need courage)
- Proof cascade (where framework gets validated)

**Output:**
- Book-level Base Story (complete narrative)
- Section-level Base Stories (one per section)
- Chapter-level Base Stories (one per chapter, with unique ID)

---

## BASE STORY STRUCTURE

### Chapter IDs

Each chapter gets unique identifier:
```
Format: [Section#]-[Chapter#]
Example: S1-C1, S1-C2, S1-C3, S2-C1, S2-C2, S3-C1, etc.
```

These IDs persist through all subsequent stages (Research, Stories, Final Draft).

### What Base Story Includes

- Complete narrative following outline structure
- Full reader journey (5-phase transformation)
- The parable beneath the framework
- Themes that echo through the book
- Tension architecture (intellectual, emotional, narrative)
- Implicit representation of all three personas
- Proof cascade (how framework gets validated)
- Voice shifts (where tone/approach changes)
- Identification of permission and proof moments

### What Base Story Does NOT Include

- Detailed research or citations
- Personal stories or case studies
- Fully polished final prose
- Exhaustive examples or applications

Base Story is the **skeletal narrative** everything else hangs on.

---

## GENERATION PROCESS

### Step 1: Extract Core Data

From Knowledge Base + TOC + Book Pitch:
- Core truth/framework
- 5-phase reader journey (Current Reality, Disruption, Revelation, Application, Transformation)
- Three personas with pain points, goals, transformations
- Voice blend weights
- Section purposes and chapter purposes
- Paragraph purposes and hooks
- Total word count and distribution

### Step 2: Write Complete Narrative

**Book-level Base Story:**
- One continuous narrative flowing from opening through closing
- Follows outline structure (sections → chapters)
- Honors 5-phase reader journey
- Weaves parable throughout
- Establishes themes
- Shows tension rising/shifting/integrating
- Implicitly honors all three persona journeys
- Incorporates voice shifts
- Length: 15-20% of final book word count

**Section-level Base Stories:**
- One per section
- Advances book's parable
- Shows 5-phase progression through section
- Demonstrates how all personas experience section
- Identifies where tension peaks
- Shows where proof accumulates
- Length: ~1,500-2,000 words per section

**Chapter-level Base Stories:**
- One per chapter with unique ID (S1-C1, S1-C2, etc.)
- Establishes chapter's core idea
- Shows role in section and book arcs
- Maps role in 5-phase journey
- Honors all persona experiences
- Identifies hooks and transitions
- Length: ~300-500 words per chapter

## OUTPUT FORMAT

### Hierarchical Structure

```
📖 BASE STORY

BOOK-LEVEL BASE STORY
[Complete narrative] — [# words]

SECTION 1: [Name]
  Section-Level Base Story — [# words]
  
  [S1-C1] CHAPTER 1: [Title]
    Chapter-Level Base Story — [# words]
  
  [S1-C2] CHAPTER 2: [Title]
    Chapter-Level Base Story — [# words]
  
  [Continue for all chapters in section]

SECTION 2: [Name]
[Repeat structure]

[Continue for all sections]
```

### Chapter ID Display

Each chapter clearly shows:
```
[S1-C1] CHAPTER 1: [Title]

Base Story:
[Chapter narrative text]

Word Count: [#] words
```

Chapter IDs are persistent and visible in all subsequent stages.

---

## QUALITY CHECKS

Before presenting Base Story, verify:

- [ ] Complete 5-phase journey shown (Current Reality → Disruption → Revelation → Application → Transformation)
- [ ] Book's parable is clear and woven
- [ ] All three personas honored (implicitly)
- [ ] Tension architecture coherent
- [ ] Major themes identified
- [ ] Voice shifts appropriate
- [ ] Follows Writing Constitution
- [ ] No research or stories included
- [ ] Narrative flows without repetition
- [ ] Opening hooks reader
- [ ] Closing leaves reader steady
- [ ] All chapters have unique IDs
- [ ] Each chapter's role in larger arc visible

If any fail: Revise before presenting.

---

## USER ACTIONS

**After Base Story generated:**

1. **Edit Document**
   - Author can edit any section/chapter Base Story directly in app
   - Changes flagged but not locked
   - Can request system regeneration of edited sections if needed

2. **Approve Base Story**
   - Locks Base Story as v1.0 FINAL
   - Status: Base Story COMPLETE ✅
   - All chapter IDs locked
   - Ready for Research & Stories stage

3. **Request Changes**
   - Specify which section(s)/chapter(s) need changes
   - Describe what needs to shift
   - System regenerates affected sections/chapters
   - Author reviews again
   - Can edit further if needed before approval
