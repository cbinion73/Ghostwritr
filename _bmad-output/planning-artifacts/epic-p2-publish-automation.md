# Epic P2 — Publish and Automation Completion

**Priority:** P2  
**Status:** Ready for refinement after P0 stabilization  
**Objective:** Close the loop from generated manuscript to publish-ready handoff with richer automation control.

## Why this epic exists

The long-term vision is not only “draft the book,” but “carry the book to a final publishable package with minimal babysitting.”

This epic closes the final product gap:

- true typesetting and publication outputs
- a fuller workflow conductor
- better multi-book/library operations
- one final publish-ready package UI

## Stories

1. `P2.1` Build Publish / Typesetting
2. `P2.2` Expand autopilot into a full workflow conductor
3. `P2.3` Upgrade dashboard and library management
4. `P2.4` Build final publish-ready package UI

## Exit criteria

- Editing hands off to a true publish stage
- the app can move a book to the next approval boundary or to a finished draft automatically
- library operations are strong enough for multiple parallel books
- final handoff is a coherent publish-ready package, not scattered exports
