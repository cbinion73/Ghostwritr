# PROMISE STAGE VALIDATION FRAMEWORK
## How GHOSTWRITR Validates Every Book Concept

**Purpose:** Integrate market validation and viability assessment into the Promise Stage as the PRIMARY GATE before proceeding to Outline.

**Owner:** Promise Stage workflow in GHOSTWRITR  
**Input:** Initial book promise statement  
**Output:** Market validation artifact + decision to proceed/refine/reject  
**Timeline:** 6 weeks (1 week per 2 phases)

---

## PROMISE STAGE WORKFLOW

### Phase 0: Promise Submission
**User:** Submits initial promise statement  
**System:** Triggers validation framework  
**Output:** Validation checklist loaded

---

### Phase 1-2: PROBLEM VALIDATION (Week 1)
**Objective:** Prove the problem is real, frequent, severe, and actively being solved

**Checklist:**
- [ ] Core problem defined in one sentence
- [ ] Frequency research completed (academic papers, reports, surveys)
- [ ] Severity quantified (time, cost, stress, career impact)
- [ ] Active problem-solving behavior documented (market evidence)
- [ ] Problem Validation Score assigned (1-5)

**Key Output:**
```
Problem Strength: [4.5/5]
Evidence Quality: [High/Medium/Low]
Next Step: [Problem is validated, proceed to Audience]
```

**Gate Decision:** If Problem Score <3.5, PAUSE and refine problem statement.

---

### Phase 2-3: AUDIENCE & ACCESS (Week 2)
**Objective:** Identify who will actually BUY this book and ensure you can reach them

**Checklist:**
- [ ] Primary buyer persona identified (job title, responsibilities, budget authority)
- [ ] Secondary/tertiary buyers mapped
- [ ] Current learning channels documented (books, courses, peers, conferences)
- [ ] Book readership likelihood assessed
- [ ] Distribution channels identified (3+ realistic options)
- [ ] Audience Clarity Score assigned (1-5)
- [ ] Audience Accessibility Score assigned (1-5)

**Key Output:**
```
Primary Buyer: [Name, role, motivation]
Learning Preference: [Books/Courses/Conferences/Peers]
Reachability: [High/Medium/Low]
Top 3 Distribution Channels: [Channel 1, Channel 2, Channel 3]
```

**Gate Decision:** If Accessibility <3.0, document how you'll reach the audience before proceeding.

---

### Phase 3-4: COMPETITIVE LANDSCAPE (Week 2-3)
**Objective:** Understand what competitors exist and where market gaps are

**Checklist:**
- [ ] 5-10 comparable books/solutions identified
- [ ] Each competitor analyzed (promise, strengths, weaknesses, reviews)
- [ ] Alternative solutions cataloged (courses, coaching, tools)
- [ ] Market density assessed (underserved/balanced/crowded)
- [ ] Clear market gaps identified
- [ ] Biggest competitive threat documented
- [ ] Competition Intensity Score assigned (1-5)

**Key Output:**
```
Top 3 Direct Competitors: [Title, Author, Year]
Market Status: [Underserved / Balanced / Crowded]
Key Gaps: [Gap 1, Gap 2, Gap 3]
Biggest Threat: [Competitor X - Why they win?]
Our Opportunity: [Where we have advantage]
```

**Gate Decision:** If market is saturated AND no clear gaps, explain how you'll differentiate before proceeding.

---

### Phase 4: DIFFERENTIATION ANALYSIS (Week 3)
**Objective:** Define what makes THIS book genuinely different and better

**Checklist:**
- [ ] Our specific promise statement extracted
- [ ] Competitor promises documented word-for-word
- [ ] Differentiation articulated in one sentence
- [ ] Reality-checked: Can we actually deliver this?
- [ ] Meaningful difference validated: Would readers get it?
- [ ] Differentiation Score assigned (1-5)

**Key Output:**
```
Our Promise:
"[Exact statement]"

Our Differentiation:
"Unlike [competitors who promise X], we [specific unique thing]"

Differentiation Strength: [5/5 - Clear and defensible]

Can readers explain why our book is better? [Yes/No]
```

**Gate Decision:** If Differentiation <2.5, REFINE promise before proceeding. Weak differentiation = book gets lost in noise.

---

### Phase 5: PRACTICAL VALUE & USABILITY (Week 4)
**Objective:** Prove readers will actually APPLY what they learn

**Checklist:**
- [ ] Reader scenario mapped (what they do after reading)
- [ ] Specific actions defined (Week 1, 2, 3, 4)
- [ ] Concrete tools/frameworks identified
- [ ] Tools are specific and templated (not abstract)
- [ ] Early wins possible within 1-2 weeks
- [ ] Implementation barriers identified
- [ ] Usability Score assigned (1-5)

**Key Output:**
```
Reader After Book: [Specific outcomes by week 1, 2, 3, 4]
Concrete Tools: [Tool 1, Tool 2, Tool 3, Tool 4, Tool 5]
Tool Specificity: [Specific with templates / Moderate / Abstract]
Early Win Timeline: [Within X days]
Implementation Barriers: [Barrier 1, Barrier 2, Barrier 3]
```

**Gate Decision:** If Usability <2.5, book is too theoretical. Add templates and step-by-step guides.

---

### Phase 6: COMPLETION LIKELIHOOD (Week 4)
**Objective:** Structure the book so busy professionals actually finish it

**Checklist:**
- [ ] Target reader time availability assessed
- [ ] Book structure reviewed (chapter length, pacing)
- [ ] Early wins identified (by chapter 3, reader knows X)
- [ ] Engagement elements documented (case studies, exercises, templates)
- [ ] Completion likelihood estimated (%)
- [ ] Why they might abandon identified
- [ ] Completion Score assigned (1-5)

**Key Output:**
```
Target Reader Time: [X hours/week available]
Chapter Structure: [X pages per chapter, Y total chapters]
Early Wins: [By chapter 3, reader will know...]
Engagement Elements: [Case studies? Workbook? Exercises?]
Estimated Completion Rate: [X%]
Reason #1 for Abandonment: [Risk 1]
```

**Gate Decision:** If Completion <3.0, restructure as workbook with shorter chapters and more interactive elements.

---

### Phase 7: SHAREABILITY & MEMORABILITY (Week 4-5)
**Objective:** Create concepts people want to talk about

**Checklist:**
- [ ] Core concept explained in one sentence
- [ ] Sticky framework/phrase identified
- [ ] Memorable vs. forgettable assessment
- [ ] Would readers recommend to peers? [Yes/No]
- [ ] Sharing potential identified (online, word-of-mouth, conferences)
- [ ] Viral potential documented
- [ ] Shareability Score assigned (1-5)

**Key Output:**
```
One-Sentence Concept: "[Sentence]"
Memorable? [Highly / Somewhat / Not really]
Sticky Framework Name: [What will readers remember?]
Recommendation Likelihood: [X%]
Best Sharing Channels: [Channel 1, Channel 2, Channel 3]
```

**Gate Decision:** If Shareability <2.0, create memorable framework name and sticky concept.

---

### Phase 8: TIMING & TRENDS (Week 5)
**Objective:** Ensure the topic is growing and will be relevant 3-5 years

**Checklist:**
- [ ] Trend analysis completed (growth/stable/declining)
- [ ] Book publishing trends researched
- [ ] Market growth projections documented
- [ ] Is this the right time to publish?
- [ ] Longevity assessed (2028? 2030?)
- [ ] Competitive timing risk identified
- [ ] Timing Score assigned (1-5)

**Key Output:**
```
Trend: [Growing rapidly / Stable / Declining]
Growth Rate: [X% year-over-year]
Book Publishing: [Growing / Stable / Declining]
Right Timing? [Yes / Too early / Too late]
Longevity (5 years): [Highly relevant / Moderate / At risk]
Competitive Timing Risk: [Threat from Heemstra, National Academies, etc.]
```

**Gate Decision:** If Timing <2.5, explain why this is the right moment despite concerns.

---

### Phase 9: COMMERCIAL VALUE (Week 5)
**Objective:** Ensure the book has real commercial viability

**Checklist:**
- [ ] TAM (Total Addressable Market) estimated
- [ ] Realistic penetration calculated
- [ ] Individual sales revenue projected (Year 1)
- [ ] Bulk/institutional sales potential estimated
- [ ] Beyond-book revenue identified (speaking, consulting, tools)
- [ ] Total revenue potential calculated
- [ ] Is book sales alone enough? [Yes/No]
- [ ] Commercial Score assigned (1-5)

**Key Output:**
```
TAM: [X potential buyers]
Estimated Year 1 Revenue: $X (individual) + $Y (bulk) = $Z
Profit After Costs: $[X]
Beyond-Book Potential: $[X] (speaking/consulting/courses)
Total 2-3 Year Potential: $[X,XXX]
Commercial Viability: [Strong / Moderate / Weak]
```

**Gate Decision:** If total potential <$50k or unclear, document business case.

---

### Phase 10: AUTHOR CREDIBILITY (Week 5)
**Objective:** Ensure readers will trust the author

**Checklist:**
- [ ] Author background assessed (years, roles, experience)
- [ ] Evidence of authority documented (research, speaking, media, audience)
- [ ] Do target readers trust this author? [Yes/No/Maybe]
- [ ] Credibility gaps identified
- [ ] Plan to address credibility gaps mapped
- [ ] Credibility Score assigned (1-5)

**Key Output:**
```
Author: [Name]
Relevant Experience: [X years in field, roles]
Authority Evidence: [Published papers? Speaking history? Audience?]
Reader Trust Level: [High / Medium / Low]
Credibility Gaps: [Gap 1, Gap 2]
Strengthening Plan: [Action 1 - timeline, Action 2 - timeline]
```

**Gate Decision:** If Credibility <2.5, author must establish credentials before launch (speaking, research, partnership with credible co-author).

---

### Phase 11: SYNTHESIS & FINAL SCORING (Week 5-6)
**Objective:** Calculate weighted score and make go/no-go decision

**Checklist:**
- [ ] All 10 dimension scores entered
- [ ] Weighted calculation completed
- [ ] Final score out of 5.0 calculated
- [ ] Interpretation assigned (Exceptional / Strong / Viable / Risky / Weak)
- [ ] Green lights identified (what's working)
- [ ] Red flags identified (what's not working)
- [ ] Recommended action selected

**Key Output:**
```
MARKET VIABILITY SCORE: __/5.0

| Dimension | Score | Status |
|-----------|-------|--------|
| 1. Problem Validation | __/5 | [Green/Yellow/Red] |
| 2. Audience Clarity | __/5 | [Green/Yellow/Red] |
| 3. Audience Accessibility | __/5 | [Green/Yellow/Red] |
| 4. Competitive Landscape | __/5 | [Green/Yellow/Red] |
| 5. Differentiation | __/5 | [Green/Yellow/Red] |
| 6. Practical Usability | __/5 | [Green/Yellow/Red] |
| 7. Readability/Completion | __/5 | [Green/Yellow/Red] |
| 8. Shareability | __/5 | [Green/Yellow/Red] |
| 9. Timing & Trends | __/5 | [Green/Yellow/Red] |
| 10. Commercial Value | __/5 | [Green/Yellow/Red] |
| 11. Author Credibility | __/5 | [Green/Yellow/Red] |
| **FINAL SCORE** | **__/5** | **[INTERPRETATION]** |

Interpretation: [Exceptional/Strong/Viable/Risky/Weak]

Recommended Action:
[ ] WRITE NOW - Score 4.0+, proceed to outline
[ ] REFINE FIRST - Score 3.5-3.99, specific improvements needed
[ ] PAUSE - Score <3.5, major rework required
```

**Gate Decision:** 
- **4.0+** → Proceed to Outline Phase immediately
- **3.5-3.99** → Conditional: Can improvements be made in 2-4 weeks? If yes, refine then proceed. If no, pause.
- **<3.5** → Do not write chapters. Revisit concept fundamentally.

---

### Phase 12: FINAL VERDICT (Week 6)
**Objective:** Direct yes/no answer with supporting reasoning

**Checklist:**
- [ ] Will people BUY this book? [Probability % & reasoning]
- [ ] Will people FINISH this book? [Probability % & reasoning]
- [ ] Will people USE what's inside? [Probability % & reasoning]
- [ ] Will people RECOMMEND it? [Probability % & reasoning]
- [ ] Overall verdict: IS THIS BOOK WORTH WRITING? [YES/REFINE/NO]

**Key Output:**
```
WILL PEOPLE BUY? [X% probability]
- Why yes: [Reason 1, Reason 2, Reason 3]
- Why no: [Barrier 1, Barrier 2, Barrier 3]

WILL PEOPLE FINISH? [X% probability]
- Why yes: [Reason 1, Reason 2, Reason 3]
- Why no: [Reason 1, Reason 2, Reason 3]

WILL PEOPLE USE IT? [X% probability]
- Why yes: [Reason 1, Reason 2, Reason 3]
- Why no: [Barrier 1, Barrier 2, Barrier 3]

WILL PEOPLE RECOMMEND? [X% probability]
- Why yes: [Reason 1, Reason 2, Reason 3]
- Why no: [Reason 1, Reason 2, Reason 3]

---

FINAL VERDICT: [YES / REFINE FIRST / NO]

Rationale: [2-3 paragraphs of supporting reasoning]

If YES: Proceed to Outline Phase immediately
If REFINE: Specific improvements required [list], then revalidate
If NO: Archive concept or revisit in [X months/years]
```

---

### Phase 13: CRITICAL IMPROVEMENTS & ACTION PLAN (Week 6)
**Objective:** If refining, create specific action plan to improve score

**Checklist (if score 3.5-3.99):**
- [ ] Single biggest weakness identified
- [ ] Fastest way to fix it documented
- [ ] Timeline assigned
- [ ] Expected score improvement calculated
- [ ] Pre-writing gates established
- [ ] Action plan created (week by week)

**Key Output:**
```
SINGLE BIGGEST WEAKNESS:
[Description of #1 limiting factor]

Why it matters: [What fails if not fixed?]

Fastest Fix:
1. [Specific action]
2. [Timeline: X weeks]
3. [Expected impact: +X points]

PRE-WRITING GATES (must all pass):
- [ ] Differentiation validated with 5+ readers
- [ ] 3-5 frameworks defined with examples
- [ ] Author credibility plan complete
- [ ] Distribution strategy mapped
- [ ] Competitive threat response documented
- [ ] Tools/templates designed
- [ ] Case studies/examples collected

ACTION PLAN (Week-by-week before writing):
Week 1: [Task 1]
Week 2: [Task 2]
Week 3: [Task 3]
Week 4: [Task 4]

Re-validation: After improvements, rescore. Target: [4.0+]
```

**Timeline:** If all gates pass, proceed to Outline Phase.

## HOW THIS CHANGES THE PROMISE STAGE

### Before (Current)
1. User writes promise
2. Wizard asks for persona/problem/promise
3. Validation system checks persona match (0-100)
4. User can refine based on scores
5. Moves to outline when they feel ready

**Problem:** Persona match is narrow metric; doesn't validate market viability, doesn't prevent weak books

### After (With Framework)
1. User writes promise
2. System loads 13-phase validation checklist
3. User completes each phase (1-2 weeks at a time)
4. System scores across 11 market dimensions
5. Gates check: Score ≥ 3.5 to proceed?
6. If YES → Moves to outline with high confidence
7. If NO → Stops and requires improvements before outline
8. Validation artifact becomes foundation for outline/positioning/marketing

**Benefit:** Every book that reaches outline has been validated against real market evidence, not just persona matching.

- Accumulate scores as user completes phases

### Phase 2: Create Evidence Database
- Store research sources for each book
- Link to public documents (academic papers, reports)
- Enable user to add custom research
- Create audit trail of validation

### Phase 3: Create Report Templates
- Auto-generate viability analysis in markdown
- Create scorecard summary
- Generate action plan if refinement needed
- Export as PDF for stakeholders

### Phase 4: Integrate with Outline Stage
- Pass validation artifact to outline
- Pre-populate persona, problem, positioning
- Use viability assessment to structure outline
- Flag lowest-scoring dimensions for content focus

**Book reaches Outline with:**
- Viability score ≥ 3.5
- Problem validation strong (4.0+)

- Distribution strategy mapped (3+ channels)
- Author credibility plan complete
- Commercial viability documented

**Result:** Books written with GHOSTWRITR have 60%+ higher finishing rate because they're validated against real markets, not hunches.
